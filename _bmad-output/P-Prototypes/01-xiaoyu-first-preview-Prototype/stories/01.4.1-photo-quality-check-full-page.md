## Goal

Build the photo quality preflight page for 小雨's first real preview flow. The page should help the user understand whether the selected photo is suitable for generation, explain the judgment in a non-black-box way, and keep every recovery path practical and low-pressure.

This is prototype-only. It must not implement real face detection, real blur detection, backend upload, AI generation, or credit charging.

---

## States Covered

- `checking`: Lightweight check is in progress; continue is disabled.
- `pass`: Photo is suitable; continue is enabled.
- `warning`: Photo can continue with risk; continue remains enabled and retake is available.
- `fail`: Photo is not suitable; continue is disabled and recovery actions are primary.
- `unavailable`: Check service failed; credit is not consumed and retry is available.

## What to Build

### HTML Structure

Replace `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.4-photo-quality-check.html` with the complete page.

Required regions:

- Mobile shell with `data-page="01.4-photo-quality-check"` and state attributes.
- Top bar with back link to `01.3-photo-upload.html`, compact brand mark, and title.
- Intro copy that frames this as a lightweight preflight, not exact face recognition.
- Photo preview card with `id="photo-quality-review-preview"`.
- Summary panel with `id="photo-quality-summary"`.
- Checklist panel with `id="photo-quality-checklist"`.
- Recovery action group with `id="photo-quality-recovery-actions"`.
- Continue button with `id="photo-quality-continue-button"`.
- Retake button with `id="photo-quality-retake-button"`.
- Compact prototype state controls for all five states.
- Toast/status region for recoverable feedback.

### JavaScript Requirements

- Load `data/demo-data.json` with a fallback for user name and quota.
- Implement `applyPhotoQualityState(stateName)`.
- Implement checklist rendering from state fixtures.
- Implement recovery actions:
  - Continue targets `01.5-generation-setup.html` only in pass/warning states.
  - Retake/reselect actions target `01.3-photo-upload.html`.
  - Retry check returns to `checking`, then automatically moves to `pass` after a short simulated delay.
  - Photo tips toggle expands/collapses practical guidance.
- Log initialization and state changes with `console.info`.

### Demo Data Requirements

Use:

- `currentUser.displayName`
- `quota.personalImageGenerationsTotal`
- `quota.personalImageGenerationsUsed`
- `photoSession.qualityCheck`

No new demo data fields are required.
