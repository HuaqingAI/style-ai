## Overview

**Page Purpose:** 小雨在保存历史结果之后，看到一个可选保存常用形象的提示。她能理解这会让下次生成少传一次照片，但不会被强迫建立档案。

**Entry Context:** 她已经在 `02.2-save-result` 保存了一个有参考价值的历史结果，当前需要决定是否把同一张形象保存为常用形象。

**Exit Action:** 选择“保存为常用形象”或“暂不保存”，都结束场景并返回首页。

**On-Page Interactions:**
- 查看刚刚保存到历史的结果摘要。
- 理解“历史结果”和“常用形象”的区别。
- 查看保存常用形象的实际好处和隐私边界。
- 保存常用形象、暂不保存，或在保存失败时重试。

## User State

**Hope:** 她希望下次继续生成或试穿时少传一次照片，同时保留自己是否建档的选择权。

**Worry:** 她担心刚才保存历史结果已经等于长期保存个人形象，或者保存常用形象后不能更换。

## Functional Requirements

- 页面必须明确“结果已经保存到历史”，本页只是单独询问常用形象。
- 页面必须使用温和、可跳过的提示，不把保存常用形象包装成必做步骤。
- 页面必须解释保存常用形象的收益：下次生成可以少传一次，可用于发型/形象建议和试穿。
- 页面必须说明边界：不保存也可以继续使用；历史结果不等于常用形象；常用形象之后可以更换。
- 页面必须提供“保存为常用形象”和“暂不保存”两个清晰动作。
- 主行动必须支持 default、saving-profile、profile-saved、skipped、save-error 五种状态。
- 保存成功或跳过后都必须提供结束路径，默认返回首页。
- 页面必须保持 Mobile-Only 单列布局，在 375px、393px、428px 宽度下无水平滚动。
