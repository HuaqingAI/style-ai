## Overview

**Page Purpose:** 让小雨用邀请码、手机号或 Google 登录进入 Style AI，并在登录前理解登录只是为了额度、历史和账号管理，不等于强制保存长期个人形象档案。

**User Situation:** 小雨晚上刷小红书时又收藏了几个风格参考，越看越不确定自己适不适合。她从定向邀请入口打开 Style AI 手机 App，想先验证“这个工具能不能让我看见自己”。

**Success Criteria:** 用户完成登录并进入首页；用户不会把登录误解为默认建立长期形象档案；用户知道首次体验有额度边界。

**Entry Points:**
- 定向邀请链接打开 App。
- 已安装用户从系统桌面启动 App。

**Exit Points:**
- 登录成功后进入 `01.2-home-dual-entry`。
- 权限或网络失败时停留本页并给出恢复方式。

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | 首次打开 | 显示品牌、登录方式、边界说明 | 输入邀请码、手机号或选择 Google 登录 |
| Loading | 正在校验邀请码、验证码或三方登录 | 登录按钮显示处理中 | 禁止重复提交 |
| Error | 邀请码无效、验证码错误、三方登录未授权、网络失败 | 显示温和具体错误 | 重新输入、重新发送、稍后再试 |
| Success | 登录成功 | 短暂过渡 | 进入首页 |

## Technical Notes

- 当前 Expo app 已有本地分析流程，但还没有登录、额度和服务端账号能力；本页需要邀请码、手机号白名单、Google OAuth/三方登录或后端账号接口。
- 首轮访问控制可以先支持本地 mock、固定邀请码、手机号白名单或 Google 账号白名单，但生产实现必须接入后端。
- 登录不应触发照片上传、档案保存或历史创建。
