## JavaScript Requirements

- Load `data/demo-data.json`.
- Render options from `styleExploration.options`.
- Keep `activeOptionId` in local state.
- Selecting an option updates active card styling and `second-style-difference-summary`.
- `applySecondStyleState(stateName)` handles `default`, `low-credit`, `submitting`, and `submit-error`.
- Primary action in default state moves to submitting state and shows a toast.
- Low-credit state disables the primary action.
- Console logging should be informational only; no console errors.
