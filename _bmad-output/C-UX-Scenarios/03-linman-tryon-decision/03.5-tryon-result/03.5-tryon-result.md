## Overview

**Page Purpose:** 林曼看到这次参考对象放到自己身上的试穿效果，并获得一个能帮助她做购买或改造决策的结论：推荐程度、适合理由、谨慎点，以及下一步动作。

**Entry Context:** 林曼已经在 `03.4-tryon-waiting` 等到生成成功。她现在最关心的不是“图片好不好看”，而是这件奶油白短外套放到她身上是否适合通勤约会两用，是否值得今晚下单。

**Exit Action:** 选择保存、加入对比、再试一个参考对象，或在看到价值后进入后续额度购买判断。

**On-Page Interactions:**
- 查看本人试穿效果图和当前参考对象。
- 查看推荐程度、适合理由、谨慎点和购买前检查项。
- 轻量反馈“像不像我”和“能不能帮我判断”。
- 保存结果、加入对比、再试一个参考图，或查看额度/购买入口。
- 在默认、已保存、已加入对比、详细说明、加载失败状态之间切换评审。

## User State

**Hope:** 她希望马上知道这件外套是否值得买，理由要具体到肩线、颜色、通勤感和搭配风险，而不是泛泛夸好看。

**Worry:** 她担心花了额度却仍然没有明确结论，或结果看起来不像自己，无法用于真实购买判断。

## Functional Requirements

- 页面必须展示本次生成结果视觉，能看出“本人形象 + 奶油白短外套”的试穿效果。
- 页面必须展示参考对象标题、来源、决策问题和本次额度扣除结果。
- 页面必须展示推荐程度，至少包含一个明确结论、两个适合理由和一个谨慎点。
- 页面必须提供购买前下一步建议，避免只输出审美判断。
- 页面必须支持轻量判断反馈：像不像本人、是否能帮助购买/造型判断。
- 页面必须支持 `default`、`saved`、`compared`、`detail`、`load-error` 五种原型状态。
- 保存结果或加入对比后，页面状态和动作文案必须更新，不重复暗示未保存。
- 保存或加入对比后必须能继续进入 `03.6-history-collection`；`03.7-credit-purchase` 尚未创建时，额度动作必须给出原型占位提示，不产生无说明的断链。
- 页面必须保持 Mobile-Only 单列布局，在 375px、393px、428px 宽度下无水平滚动。
