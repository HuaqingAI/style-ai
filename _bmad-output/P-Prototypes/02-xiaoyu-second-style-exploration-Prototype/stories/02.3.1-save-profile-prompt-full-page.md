## Objects

- `save-profile-header`: mobile header with brand, context label, back link to `02.2-save-result.html`, and home link.
- `save-profile-hero`: low-pressure prompt asking whether to save the current image as a common profile.
- `save-profile-context`: saved history result context, making clear that history save already succeeded.
- `save-profile-benefits`: benefit list for fewer uploads, cross-scenario reuse, and later replacement.
- `save-profile-boundary-note`: privacy and product-boundary note separating history result, common profile, and skipped choice.
- `save-profile-primary-action`: save-as-common-profile button with default/saving/saved/error behavior.
- `save-profile-skip-action`: skip action that ends the scenario without saving a common profile.
- `save-profile-state-feedback`: status feedback for success, skipped, or error states.
- `save-profile-state-controls`: local reviewer controls for default, saving-profile, profile-saved, skipped, and save-error.

## JavaScript Requirements

- Load `data/demo-data.json`.
- Use `commonProfilePrompt` from demo data when available.
- Keep `currentState` in local state.
- `applySaveProfileState(stateName)` handles `default`, `saving-profile`, `profile-saved`, `skipped`, and `save-error`.
- Primary action:
  - `default` and `save-error`: transitions to `saving-profile`, then `profile-saved`.
  - `saving-profile`: does nothing.
  - `profile-saved` and `skipped`: navigates to `/01-xiaoyu-first-preview-Prototype/01.2-home-dual-entry.html`.
- Skip action:
  - `default` and `save-error`: transitions to `skipped`.
  - Disabled during `saving-profile`.
- State controls must allow manual state switching.
- Console logging should be informational only; no console errors.

## Agent-Verifiable Acceptance Criteria

- `lang="zh-CN"` exists.
- All required Object IDs exist.
- Default state shows both `save-profile-primary-action` and `save-profile-skip-action`.
- Clicking primary save enters `profile-saved` after a saving state.
- Clicking skip enters `skipped`.
- `saving-profile` disables save and skip actions.
- `save-error` displays a visible error message and keeps recovery actions available.
- `profile-saved` and `skipped` set the primary action to return home.
- 375px, 393px, and 428px widths have no horizontal overflow.
- Browser console has no errors.

---

## User-Evaluable Acceptance Criteria

- The prompt feels optional rather than coercive.
- It is clear that the result is already saved to history.
- It is clear why saving a common profile helps future generation.
- It is clear that not saving is a valid choice.
- The privacy and profile boundary copy feels consistent with earlier pages.
