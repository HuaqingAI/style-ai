# Story 01.3.1: Photo Upload - Full Page

**Page**: 01.3 photo-upload  
**Section**: Full page vertical slice  
**Complexity**: Medium  
**Estimated Time**: 2 hours

---

## Goal

Build the authenticated photo upload page for 小雨's first real preview flow. The page must make the photo-use boundary clear before action, provide camera and album entry points, show a selected-photo confirmation state, and keep permission or read failures recoverable on the same page. The underlying photo intake pattern must stay reusable for personal style advice and try-on flows.

This is prototype-only. It must not implement native permissions, real file upload, backend storage, AI generation, or persistent profile creation.

## States Covered

- `Empty`: upload area and picker actions are visible; continue action is unavailable.
- `Permission denied`: permission explanation and settings guidance are visible; user remains on page.
- `Selected`: simulated image preview, source, reselect action, and continue button are visible.
- `Upload error`: specific read failure explanation is visible; user can retry.

## What to Build

### HTML Structure

Create `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.3-photo-upload.html` using the mobile prototype visual language from `01.2-home-dual-entry.html`.

Required regions:

- Mobile shell with `data-page="01.3-photo-upload"`.
- Top bar with a back link to `01.2-home-dual-entry.html`, compact brand mark, and page title.
- Boundary note section with `id="photo-upload-boundary-note"` and an expandable detail area.
- Picker panel with `id="photo-upload-picker"`, `photo-upload-camera-button`, and `photo-upload-album-button`.
- Preview card with `id="photo-upload-selected-preview"`.
- Continue button with `id="photo-upload-continue-button"`.
- Photo tips section with `id="photo-upload-photo-tips"`.
- Compact prototype state controls for agent verification.
- Toast/status region for recoverable feedback.

### JavaScript Requirements

- Load `data/demo-data.json` with a safe fallback for user name and photo boundary flags.
- Implement `applyPhotoUploadState(stateName, options)`.
- Implement click behavior:
  - Camera action enters selected state with source `camera`.
  - Album action enters selected state with source `album`.
  - Continue action navigates to `01.4-photo-quality-check.html` only when state is selected.
  - Reselect action returns to empty state.
  - Boundary note toggles expanded/collapsed details.
  - Permission denied and upload error states can be triggered from compact prototype controls.
- Log initialization and state changes with `console.info`.

### Reuse Contract

- `intent`: `personal_style_advice` for this page instance.
- `reusableFor`: `personal_style_advice`, `try_on`.
- `photoScope`: `session_photo`.
- `persistentProfileRequiresExplicitSave`: `true`.
- `nextTarget`: `01.4-photo-quality-check.html`.

### Demo Data Requirements

Use:

- `currentUser.displayName`
- `photoSession.privacyBoundary.temporaryPhotoUse`
- `photoSession.privacyBoundary.requiresExplicitSaveAfterResult`

No new demo data fields are required.
