**Section**: Full page vertical slice

## Goal

Build the async generation waiting page for Xiaoyu's first real preview flow. The page should make the generation task feel alive and recoverable: show status, progress, credit handling, plain-language processing steps, and clear recovery paths for timeout or failure.

This is prototype-only. It must not implement real image generation, real backend polling, or real credit charging.

---

## States Covered

- `queued`: Task was created and is waiting to start.
- `generating`: Model is processing the preview.
- `finalizing`: Result returned and copy is being organized.
- `success`: Result is ready; user can enter result page.
- `timeout`: Waiting exceeded the expected threshold; user can keep waiting or leave.
- `failed`: Generation failed; credit is not deducted and retry is available.

## What to Build

### HTML Structure

Create `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.6-generation-waiting.html` with:

- Mobile shell with `data-page="01.6-generation-waiting"` and `data-state`.
- Top bar with back link to `01.5-generation-setup.html`, brand mark, and title.
- Intro copy that frames waiting as a transparent async step.
- Status panel with `id="generation-waiting-status"`.
- Credit note with `id="generation-waiting-credit-note"`.
- Detail list with `id="generation-waiting-detail-list"`.
- Recovery action group with `id="generation-waiting-recovery-actions"`.
- Compact prototype state controls for all six states.
- Toast/status region for local feedback.

### JavaScript Requirements

- Load `data/demo-data.json` with fallback user, quota, generation id, status, progress, and estimated seconds.
- Parse query parameters `taskId`, `style`, and `state`.
- Implement `applyGenerationWaitingState(stateName)`.
- Implement local fixtures for queued, generating, finalizing, success, timeout, and failed.
- Update:
  - progress value and visual ring
  - status title/copy
  - credit note title/copy/tone
  - detail list status
  - recovery action visibility
  - prototype state control active styling
- Wire actions:
  - Continue waiting returns to generating.
  - View result targets `01.7-image-result.html`.
  - Retry targets `01.5-generation-setup.html`.
  - Change photo targets `01.3-photo-upload.html`.
  - Home targets `01.2-home-dual-entry.html`.
- Log initialization, state changes, and action clicks with `console.info`.

### Demo Data Requirements

Use:

- `currentUser.displayName`
- `quota.personalImageGenerationsTotal`
- `quota.personalImageGenerationsUsed`
- `generation.id`
- `generation.status`
- `generation.progress`
- `generation.estimatedSeconds`

No new demo data fields are required.
