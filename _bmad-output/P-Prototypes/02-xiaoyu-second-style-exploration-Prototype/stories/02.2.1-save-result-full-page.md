## Objects

- `save-result-header`: mobile header with brand, context label, back link to `02.1-second-style-exploration.html`, and home link.
- `save-result-hero`: concise page intro that the second result is ready and this step saves a useful history result.
- `save-result-preview`: second result preview using the existing first-result image asset as a stylized placeholder, with second-result labels and confidence signals.
- `save-result-comparison`: comparison between first and second result including difference, useful-for, and caution notes.
- `save-result-choice-group`: radio-like buttons for saving the second result or the first result; second result is default.
- `save-result-value-note`: explanation that history save is for later comparison and is separate from common profile.
- `save-result-primary-action`: main save button with default/saving/saved/error behavior.
- `save-result-secondary-actions`: links to generate another result or return to the first result.
- `save-result-state-controls`: local reviewer controls for default, saving, saved, and save-error.

## JavaScript Requirements

- Load `data/demo-data.json`.
- Use `saveResult.defaultSaveTarget` and `saveResult.choices` from demo data.
- Keep `activeSaveTarget` in local state.
- `setSaveTarget(targetId)` updates active card styling, preview labels, save note, and button copy.
- `applySaveResultState(stateName)` handles `default`, `saving`, `saved`, and `save-error`.
- Primary action:
  - `default` and `save-error`: transitions to `saving`, then `saved`.
  - `saving`: does nothing.
  - `saved`: navigates to `02.3-save-profile-prompt.html`.
- State controls must allow manual state switching.
- Console logging should be informational only; no console errors.

- The page feels like a natural continuation after the second generation completes.
- It is clear what is being saved and why.
