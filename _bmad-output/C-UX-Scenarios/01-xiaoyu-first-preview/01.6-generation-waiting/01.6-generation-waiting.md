## Overview

**Page Purpose:** 在生成等待期间维持信任感，展示进度、默认生成方向、本次额度处理和失败恢复路径。

**User Situation:** 小雨已经消耗时间和心理预期，正在等第一次真实预览。如果等待页只显示空转 loading，她会焦虑结果是否会失败或扣费。

**Success Criteria:** 用户愿意等待生成完成；用户知道失败不会扣除或会自动恢复；生成完成后进入结果页。

**Entry Points:**
- `01.3-photo-upload` 选择合格或 warning 照片后，直接创建默认生成任务。
- `01.5-generation-setup` 可作为后续/高级路径来源，但不属于首次 happy path。

**Exit Points:**
- 生成成功进入 `01.7-image-result`。
- 生成失败进入可重试状态。

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Queued | 任务创建后等待执行 | 显示排队状态 | 等待或返回首页 |
| Generating | 模型处理中 | 显示生成中状态 | 等待 |
| Finalizing | 结果已返回，正在整理分析 | 显示整理中状态 | 等待 |
| Success | 结果准备完成 | 短暂完成状态 | 自动进入结果页 |
| Timeout | 超过合理等待时间 | 说明还在处理或可稍后查看 | 查看历史、继续等待 |
| Failed | 生成失败 | 明确不扣额度或退回额度 | 重试、换照片 |

## Technical Notes

- 需要 generation task 状态模型：`queued`, `processing`, `finalizing`, `succeeded`, `failed`, `timeout`。
- 当前代码 `analyzeImage` 是一次请求直接返回 JSON 分析；DD-001 的 image2 生成更适合异步任务接口。
- IMP-001 首轮任务应带 `defaultGenerationIntent: daily_judgment_safe_direction` 和 `source: first_preview_auto_default`。
- 等待页应能从后台恢复：用户切出 App 后回来，仍可查看任务状态。
