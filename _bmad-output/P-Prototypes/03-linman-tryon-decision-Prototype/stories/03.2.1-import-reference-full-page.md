# 03.2.1 Import Reference Full Page

## Purpose

Build `03.2-import-reference.html` as a mobile prototype draft for importing an external reference image before try-on setup.

- `_bmad-output/C-UX-Scenarios/03-linman-tryon-decision/03-linman-tryon-decision.md`

## Objects

- `obj-import-header`: top navigation with back path to `03.1-tryon-entry.html`.
- `obj-import-hero`: concise promise that this page imports the reference only; quota is confirmed later.
- `obj-import-source-options`: three source buttons for album/screenshot, current page share, and sample reference.
- `obj-reference-preview`: preview state container for empty, importing, imported, and error.
- `obj-reference-details`: imported reference metadata and decision question.
- `obj-import-recovery`: no-quota/no-selection/failure reassurance and recovery copy.
- `obj-confirm-reference-action`: primary action, disabled until imported, then links to `03.3-tryon-setup.html`.
- `obj-prototype-state-controls`: local reviewer controls for all states.

## State Behavior

- `empty`: no selected reference; primary action is disabled.
- `importing`: show progress message and keep primary action disabled.
- `imported`: show the selected candidate; primary action becomes available.
- `error`: show failure message and retry paths.

## Tailwind / Layout Requirements

- Mobile-only single column, max width `428px`.
- Use dark Style AI surface `#111111`, ivory text `#FAFAF7`, spark accents `#7B52FA` and `#9D7AFF`.
- Cards should use `rounded-lg`, restrained borders, no nested decorative card stacks.
- Text must fit at 375px without horizontal scroll.

## JavaScript Requirements

- Implement `setImportState(state, candidateId)`.
- Implement import buttons that transition through `importing` and then `imported`.
- Implement an error simulation button through prototype controls.
- Update `data-state` on the app root.
- Update primary action disabled state and `href`.

## Acceptance Criteria

Agent-verifiable:
- All object IDs exist.
- Four states can be triggered.
- Imported state enables `obj-confirm-reference-action`.
- Empty/importing/error states disable forward navigation.
- No console errors at load and during state switching.
- No horizontal overflow at 375px, 393px, and 428px.

User-evaluable:
- It is clear that the user is choosing a reference image, not generating yet.
- It is clear which object is being imported and what decision it supports.
- The page feels fast enough for an immediate shopping or styling decision moment.
