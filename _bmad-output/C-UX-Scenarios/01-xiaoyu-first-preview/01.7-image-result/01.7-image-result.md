## Overview

**Page Purpose:** 让小雨看到足够像自己的风格预览，并获得温和、具体、可判断的推荐理由、谨慎点和下一步动作。

**User Situation:** 小雨最关心的不是“AI 夸我”，而是这张结果是否像自己、是否能帮助她判断现实里能不能尝试。

**Success Criteria:** 用户完成首次真实预览；用户能评价“像不像自己”和“能不能用于判断”；用户至少有一个继续动作：换风格、保存结果、保存为常用形象或反馈。

**Entry Points:**
- `01.6-generation-waiting` 生成成功。
- 历史页重新打开首次结果。

**Exit Points:**
- 继续换风格进入 Scenario 02。
- 保存结果或保存常用形象。
- 提交轻量反馈。
- 返回首页。

## Reference Materials

**Strategic Foundation:**
- `_bmad-output/B-Trigger-Map/trigger-map.md` - 结果不只给图，还要给判断：推荐、谨慎点、原因和反馈。
- `_bmad-output/A-Product-Brief/product-brief.md` - MVP primary metric 是“像自己、可用于判断”。

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | 结果可用 | 图、分析、下一步完整展示 | 保存、换风格、反馈 |
| Partial result | 图可用但分析缺失 | 图优先展示，分析显示稍后补全 | 保存、刷新分析 |
| Load error | 打开历史结果失败 | 显示恢复说明 | 重试、返回历史 |
| Feedback submitted | 用户提交反馈 | 感谢提示，保留继续探索动作 | 换风格或返回 |

## Technical Notes

- 当前 app 的 `AnalysisResult` 只有评分、summary、aspects、suggestions；DD-001 需要结果模型支持生成图 URL、推荐方向、谨慎点、相似度/可判断反馈。
- 结果保存到历史不应自动保存为常用形象。
- 反馈数据需要结构化字段，至少包含 `looks_like_me`, `useful_for_judgment`, `issue_tags`, `free_text`。
