## Overview

**Page Purpose:** 林曼提交一次即时试穿/试造型生成后，看到任务进度、正在处理的内容、本次额度处理规则，以及失败或等待较久时该怎么恢复。

**Entry Context:** 林曼已经在 `03.3-tryon-setup` 确认参考图、本人形象和本次消耗，并点击开始生成。她正处在“已经付出等待成本，但还没有看到结果”的敏感节点。

**Exit Action:** 生成完成后进入 `03.5-tryon-result`。结果页尚未创建时，原型以占位提示说明下一步。

**On-Page Interactions:**
- 查看当前生成状态、进度百分比、预计剩余时间和任务编号。
- 查看本次额度处理说明：成功准备好结果后扣除 1 个生成额度；失败不扣除或退回。
- 查看处理步骤：读取参考图、匹配本人形象、生成试穿预览、整理判断提示。
- 在排队、生成中、整理中、完成、等待较久、失败状态之间切换评审。
- 在完成状态进入结果页；在失败状态重试或返回设置页。

## User State

**Hope:** 她希望等待过程短、确定，并且知道系统真的在处理“刚才那张参考图”和“她的本人形象”。

**Worry:** 她担心流程卡住、重复扣额度、失败后不知道能不能退回或重试，也担心等待太久错过当下的购买/造型决策窗口。

## Functional Requirements

- 页面必须展示生成进度状态，包括状态标签、进度百分比、预计时间和任务编号。
- 页面必须展示本次试穿对象摘要，至少包含参考图标题、来源、本人形象和本次判断问题。
- 页面必须展示额度处理说明：成功生成后扣除 `1 个生成额度`，失败不扣或退回。
- 页面必须展示处理步骤，并用不同状态标识已完成、处理中、等待、未完成。
- 页面必须支持 `queued`、`generating`、`finalizing`、`success`、`timeout`、`failed` 六种原型状态。
- `success` 状态的主行动必须指向 `03.5-tryon-result.html`；如果该页面未创建，原型必须给出明确占位提示。
- `failed` 状态必须说明失败未消耗额度，并提供重试/返回设置的恢复路径。
- 页面必须保持 Mobile-Only 单列布局，在 375px、393px、428px 宽度下无水平滚动。

## Object Inventory

| Object ID | Name | Role |
|-----------|------|------|
| `obj-waiting-header` | 页面顶部导航 | 标识当前处于试穿生成等待页，并可返回 03.3。 |
| `obj-waiting-hero` | 等待说明区 | 安抚用户等待情绪，说明正在生成她的试穿预览。 |
| `obj-waiting-status` | 生成状态面板 | 展示状态标签、进度、预计时间、任务编号和当前说明。 |
| `obj-waiting-context-summary` | 本次任务摘要 | 展示参考对象、本人形象、试穿意图和决策问题。 |
| `obj-waiting-credit-policy` | 额度处理说明 | 明确成功后扣 1 个额度，失败不扣或退回。 |
| `obj-waiting-processing-steps` | 处理步骤列表 | 展示读取参考图、匹配本人形象、生成预览、整理判断提示的状态。 |
| `obj-waiting-recovery-actions` | 等待/失败恢复动作 | 根据状态展示继续等待、查看结果、重试、返回设置等动作。 |
| `obj-waiting-state-controls` | 原型状态控制 | 方便评审六种等待页状态。 |
