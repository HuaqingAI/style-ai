# Story 01.5.1: Generation Setup - Full Page

**Page**: 01.5 generation-setup  
**Section**: Full page vertical slice  
**Complexity**: Medium  
**Estimated Time**: 2 hours

---

## Goal

Build the generation setup page for Xiaoyu's first real preview flow. The page should keep the pre-generation decision lightweight: confirm the current photo, offer a small set of understandable style directions, make the credit cost explicit, and let the user start generation without needing expert prompt controls.

This is prototype-only. It must not implement real image generation, real backend task creation, or real credit charging.

## Objects Covered

| Object ID | Type | Label / Content | Behavior |
|-----------|------|-----------------|----------|
| `generation-setup-context` | Summary | Current photo thumbnail and reassurance copy | Confirms this generation uses the selected session photo |
| `generation-setup-style-selector` | Selector | 自然通勤、温柔清爽、利落轻熟、甜酷尝试、先由 AI 推荐 | One selected direction at a time; default is AI recommend |
| `generation-setup-style-ai-recommend` | Option | 先由 AI 推荐 | Default selected option; lets the system choose a safe first direction |
| `generation-setup-credit-summary` | Credit panel | 本次预计消耗 1 次生成额度。生成失败不会扣除。 | Shows available quota and no-credit recovery |
| `generation-setup-start-action` | Button | 开始生成我的预览 | Disabled for no-credit/submitting; creates a simulated task when available |
| `generation-setup-preference-note` | Optional note | 可选一句偏好 | Lets user add one short non-required preference |

---

## States Covered

- `default`: Photo and credit are available; AI recommend is selected; start is enabled.
- `no-credit`: Available quota is 0; start is disabled and credit entry is visible.
- `submitting`: A task is being created; start is disabled and repeated submission is blocked.
- `submit-error`: Task creation failed; no credit is deducted and retry is available.

## What to Build

### HTML Structure

Replace `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.5-generation-setup.html` with the complete page.

Required regions:

- Mobile shell with `data-page="01.5-generation-setup"` and state attributes.
- Top bar with back link to `01.3-photo-upload.html`, compact brand mark, and title.
- Intro copy that frames style selection as a low-pressure direction choice.
- Photo context card with `id="generation-setup-context"`.
- Style selector with `id="generation-setup-style-selector"`.
- AI recommend option with `id="generation-setup-style-ai-recommend"` and default selected state.
- Optional note with `id="generation-setup-preference-note"`.
- Credit panel with `id="generation-setup-credit-summary"`.
- Start button with `id="generation-setup-start-action"`.
- Compact prototype state controls for all four states.
- Toast/status region for recoverable feedback.

### JavaScript Requirements

- Load `data/demo-data.json` with fallback user, quota, photo, and generation data.
- Implement `applyGenerationSetupState(stateName)`.
- Implement `selectStyleDirection(styleId)`.
- Implement local state fixtures for default, no-credit, submitting, and submit-error.
- Keep AI recommend selected by default.
- Update credit text based on demo quota or no-credit override.
- Handle start generation:
  - If no credit, do nothing except show feedback.
  - If available, set submitting state, create simulated `taskId`, and navigate to `01.6-generation-waiting.html`.
  - If submit-error state is selected, allow retry.
- Log initialization, style changes, state changes, and simulated submit with `console.info`.

### Demo Data Requirements

Use:

- `currentUser.displayName`
- `quota.personalImageGenerationsTotal`
- `quota.personalImageGenerationsUsed`
- `generation.id`
- `generation.selectedStyleDirection`

No new demo data fields are required.
