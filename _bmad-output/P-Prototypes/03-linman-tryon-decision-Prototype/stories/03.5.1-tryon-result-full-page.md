# 03.5.1 Tryon Result Full Page

## Purpose

Build `03.5-tryon-result.html` as a mobile prototype draft for the completed try-on result after Linman's generated preview succeeds.

## Specification References

- `_bmad-output/C-UX-Scenarios/03-linman-tryon-decision/03-linman-tryon-decision.md`
- `_bmad-output/C-UX-Scenarios/03-linman-tryon-decision/03.5-tryon-result/03.5-tryon-result.md`
- `_bmad-output/P-Prototypes/03-linman-tryon-decision-Prototype/data/demo-data.json`

## Objects

- `obj-result-header`: top navigation with back path to `03.4-tryon-waiting.html`.
- `obj-result-hero`: concise result completion copy with credit status and decision framing.
- `obj-result-viewer`: primary generated try-on visual for Linman wearing the cream short jacket.
- `obj-result-reference-summary`: imported reference metadata and original decision question.
- `obj-result-fit-summary`: recommendation degree, reasons, caution, and purchase checks.
- `obj-result-judgment-feedback`: quick feedback buttons for "像不像我" and "能不能帮我判断".
- `obj-result-next-actions`: save, compare, retry/new reference, and credit-related next actions.
- `obj-result-state-controls`: local reviewer controls for five page states.

## State Behavior

- `default`: result is available and not yet saved or compared.
- `saved`: result is saved to history; save chip and action copy update.
- `compared`: result is saved and added to comparison; comparison note appears.
- `detail`: more purchase check details are expanded.
- `load-error`: result content is hidden and recovery actions are shown.

## Layout Requirements

- Mobile-only single column, max width `428px`.
- Use the same Tailwind CDN setup and dark Style AI surface as `03.2` through `03.4`.
- Use stable aspect ratio for the result viewer so the page does not shift at 375px.
- Cards use `rounded-lg`, light borders, and compact text scale.
- Avoid nested decorative cards and avoid horizontal overflow.
- The primary visual must appear before analysis cards.

## JavaScript Requirements

- Implement `setResultState(state)`.
- Implement `saveResult()`, `addToCompare()`, `submitJudgment(field, value)`, and `showResultToast(message)`.
- Update `data-state`, `data-saved`, `data-compared`, and `data-detail-open` on `#app`.
- Use local data from `tryOnResult` in `data/demo-data.json` with a complete fallback object in the page.
- Route saved/compared result actions into `03.6-history-collection.html`; keep the `03.7` credit action as a toast placeholder until that page is built.
- Expose `window.__STYLE_AI_PROTOTYPE__` with state setter and action helpers for automated verification.

## Acceptance Criteria

Agent-verifiable:
- All object IDs exist.
- Five states can be triggered.
- Save and compare actions update app dataset and visible chips.
- Quick judgment feedback can be selected without form submission.
- Load-error state hides result content and shows recovery actions.
- `03.4` success action can reach `03.5-tryon-result.html`.
- Saved/compared result actions can reach `03.6-history-collection.html`.
- No console errors at load and during state switching.
- No horizontal overflow at 375px, 393px, and 428px.

User-evaluable:
- The result clearly supports Linman's immediate purchase/try-on decision.
- Recommendation reasons feel specific to the jacket, body/profile match, and decision question.
- Caution copy is practical without sounding judgmental.
- Next actions make sense after seeing value from one consumed credit.
