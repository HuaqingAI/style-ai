## Overview

**Page Purpose:** 让小雨在首次生成前选择一个低压力的主流风格方向，并清楚看到本次生成会消耗多少额度。

**User Situation:** 小雨希望先看一个“能判断”的方向，不想在复杂参数里做专业选择。她需要简单选项和温和解释。

**Success Criteria:** 用户选择一个风格方向并开始生成；用户知道本次消耗和失败处理；用户没有被复杂参数拖住。

**Entry Points:**
- `01.4-photo-quality-check` 照片检查通过后进入。

**Exit Points:**
- 点击开始生成后进入 `01.6-generation-waiting`。
- 返回照片质量页或重新选择照片。

## Reference Materials

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | 有可用照片和额度 | 显示风格选择与开始按钮 | 选择风格、开始生成 |
| No credit | 额度不足 | 开始按钮禁用，解释剩余额度 | 查看额度或返回 |
| Submitting | 正在创建任务 | 按钮处理中 | 禁止重复提交 |
| Submit error | 任务创建失败 | 显示不扣额度的错误 | 重试或返回 |

## Technical Notes

- 当前代码里的 `AnalysisCategory` 是发型/穿搭/妆容/配饰点评分类；DD-001 需要新增或改造为“个人形象生成风格方向”。
- 生成任务应有 task id，用于等待页轮询或订阅状态。
- 失败不扣额度规则需要后端支持，前端只展示状态。
