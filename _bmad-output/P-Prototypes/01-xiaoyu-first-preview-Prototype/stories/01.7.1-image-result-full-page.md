# Story 01.7.1: Image Result - Full Page

**Page**: 01.7 image-result  
**Section**: Full page vertical slice  
**Complexity**: Medium  
**Estimated Time**: 2.5 hours

---

## Goal

Build the first generated image result page for Xiaoyu's first real preview flow. The page should prioritize the result image, then explain why this direction is usable for judgment without turning the experience into an appearance score. It should give clear next actions: explore another style, save the result, optionally save the image as a common profile, or leave feedback.

This is prototype-only. It must not implement real image generation, real history storage, profile persistence, or backend feedback submission.

Design decision after qualitative review: the page should model **single-generation fixed atlas + App-composed display**. AI generates one fixed visual atlas/contact sheet for cost control; server-side crop metadata turns it into a primary result preview and optional analysis tiles; the App renders judgment, copy, actions, and feedback natively on one result page.

## States Covered

- `default`: Result image and analysis are available.
- `partial-result`: Image is available, analysis is still being completed.
- `load-error`: A historical result failed to load and offers recovery.
- `feedback-submitted`: Feedback was submitted and continuation actions remain available.

---

## What to Build

### HTML Structure

Create `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.7-image-result.html`.

Required regions:

- Mobile shell with `data-page="01.7-image-result"` and state attributes.
- Top bar with back/home navigation, compact brand mark, title, and saved status.
- Result viewer with `id="image-result-viewer"` using a cropped primary result preview, not a full long report image.
- Quick judgment strip with `id="image-result-judgment-strip"` directly below the result viewer.
- Analysis card with `id="image-result-fit-summary"` containing recommendation, reason bullets, and caution note.
- Required content objects `image-result-recommended-direction` and `image-result-caution-note`.
- Next action group with required explore and save buttons.
- Optional detailed analysis area with `id="image-result-analysis-detail"` that uses cropped atlas tiles and can be collapsed by default.
- Common profile prompt with save/skip/learn-difference local actions.
- Seed feedback panel or expansion with structured controls and submit action.
- Compact prototype state controls for the four states.
- Toast/status region for recoverable feedback.

### JavaScript Requirements

- Load `data/demo-data.json` with fallback user and result data.
- Support a result display fixture with `previewImageUrl`, `analysisAtlasUrl`, `detailTileUrls`, `cropMetadata`, `recommendedDirection`, `reasonBullets`, and `cautionNote`.
- Implement `applyImageResultState(stateName)`.
- Implement local save-result and save-profile interactions.
- Implement structured feedback collection for:
  - `looks_like_me`
  - `useful_for_judgment`
  - `issue_tags`
  - `free_text`
- Transition to `feedback-submitted` after feedback submission.
- Keep explore action navigable even after feedback.
- Log initialization, state changes, save actions, profile prompt actions, and feedback payload with `console.info`.
