# Story 01.2.1: Home Dual Entry - Full Page

**Page**: 01.2 home-dual-entry  
**Section**: Full page vertical slice  
**Complexity**: Medium  
**Estimated Time**: 2 hours

---

## Goal

Build the authenticated home page that presents two product entry paths while clearly prioritizing “生成我的形象方案” for 小雨’s first real preview flow. The page must show remaining credit, keep try-on visible without pulling the user out of DD-001, and prevent an empty first-run home state.

This is prototype-only. It must not implement production billing, real history retrieval, account settings, photo upload, or Scenario 03.

---

## States Covered

- `First-time`: no history and no persistent profile; primary card is the clear first path.
- `Returning`: recent result prompt replaces the empty history copy.
- `Low credit`: credit snapshot shows 0, primary action explains quota instead of implying generation can start.
- `Feature unavailable`: secondary try-on entry is visible and clearly labeled as not open in this seed-test slice.

---

## What to Build

### HTML Structure

Create `_bmad-output/P-Prototypes/01-xiaoyu-first-preview-Prototype/01.2-home-dual-entry.html` using the existing mobile prototype visual language from `01.1-start-login.html`.

Required regions:

- Mobile shell with `data-page="01.2-home-dual-entry"`.
- Authenticated header with `id="home-dual-entry-header"`.
- Credit chip with `id="home-dual-entry-credit-snapshot"`.
- Primary entry action card with `id="home-dual-entry-primary-card"` and visible button object `home-dual-entry-primary-button`.
- Secondary entry action card with `id="home-dual-entry-secondary-card"`.
- History/empty area with `id="home-dual-entry-empty-history-prompt"`.
- Compact prototype state controls for agent verification.

### JavaScript Requirements

- Load `data/demo-data.json` with a safe fallback.
- Compute available personal-image quota.
- Implement `applyHomeState(stateName)`.
- Implement click behavior:
  - Primary action targets `01.3-photo-upload.html` when credit exists.
  - Primary action shows quota feedback in low-credit state.
  - Secondary action shows “试穿/试造型会在种子测试下一批开放。”
  - Credit/history/profile shortcuts show non-blocking prototype feedback.
- Log initialization and state changes with `console.info`.

### Demo Data Requirements

Use:

- `currentUser.displayName`
- `quota.personalImageGenerationsTotal`
- `quota.personalImageGenerationsUsed`
- `copy.firstPreviewPromise`

No new demo data fields are required.
