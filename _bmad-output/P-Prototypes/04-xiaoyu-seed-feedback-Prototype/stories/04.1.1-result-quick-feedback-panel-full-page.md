## Objects Implemented

| Object ID | Type | Behavior |
|-----------|------|----------|
| `obj-feedback-header` | Header | Back and home navigation, page title |
| `obj-result-context` | Result context | Shows current result image and result summary |
| `obj-feedback-entry` | Inline entry | Opens the feedback panel |
| `obj-feedback-panel` | Panel | Contains all lightweight feedback controls |
| `obj-likeness-choice-group` | Segmented choices | Single-select likeness signal |
| `obj-judgment-choice-group` | Segmented choices | Single-select usefulness signal |
| `obj-issue-tags` | Tag group | Multi-select dynamic issue tags |
| `obj-issue-category-group` | Category group | Single-select issue category |
| `obj-feedback-note` | Textarea | Optional free-text detail |
| `obj-submit-feedback-action` | Button | Validates and submits feedback |
| `obj-feedback-validation` | Alert | Appears when submitting with no signal |
| `obj-feedback-success` | Success block | Appears after submission |
| `obj-feedback-next-actions` | Navigation | Continue and retry links after submission |
| `obj-feedback-state-controls` | Prototype controls | Applies review states |

---

## States

| State | Expected Behavior |
|-------|-------------------|
| `default` | Personal result context, feedback panel collapsed |
| `open` | Feedback panel expanded |
| `validation-error` | Panel expanded and validation alert visible |
| `submitting` | Submit button disabled with loading copy |
| `submitted` | Thanks state and next actions visible |
| `tryon-context` | Try-on result context with try-on-specific issue tags |

## Production Notes

- Use this as the interaction contract for embedding a feedback panel into existing production result pages.
- Feedback payload should include source result ID, result type, likeness, judgment usefulness, quick tags, category, optional text, and timestamp.
- Submission should not block continue/retry actions.
