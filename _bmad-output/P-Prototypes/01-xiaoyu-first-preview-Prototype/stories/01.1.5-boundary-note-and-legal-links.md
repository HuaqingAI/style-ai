## Common Issues & Fixes

### Issue: Boundary Note Sounds Like a Legal Warning

**Symptom**: The page feels heavier or more anxious at the bottom.  
**Cause**: Long privacy/legal copy or too much emphasis.  
**Fix**: Keep one concise sentence and subdued visual weight.

### Issue: Legal Links Look Required Before Login

**Symptom**: User thinks they must read or accept terms before continuing.  
**Cause**: Links styled like a checkbox, alert, or blocking confirmation.  
**Fix**: Keep them as compact footer links only; do not add a checkbox.

### Issue: Boundary Copy Reintroduces Seed-Test Framing

**Symptom**: Page feels like a special beta access page again.  
**Cause**: Copy over-explains invite-only rollout or test restrictions.  
**Fix**: Keep access control out of this note; use normal product login framing.

### Issue: Footer Pushes Primary Action Too Far

**Symptom**: On small screens, the main login action feels crowded or displaced.  
**Cause**: Boundary note or footer uses too much vertical space.  
**Fix**: Keep text at `text-xs`, one paragraph, and compact footer spacing.

## Design Notes

Visual requirements:

- Use the existing dark shell, warm off-white text, and restrained purple accents.
- The boundary note should be visibly secondary to the login action.
- No nested cards, no alert-style banner, no decorative spark/orb treatment.
- Radius remains 8px.

UX considerations:

- Login is for quota, history, and account management.
- Saving as a reusable personal profile is a later explicit decision after generation.
- Access control remains handled by targeted rollout, invite code, phone whitelist, Google account whitelist, or backend rules, not prominent UI copy.
- This note should reduce anxiety without creating a new task.
