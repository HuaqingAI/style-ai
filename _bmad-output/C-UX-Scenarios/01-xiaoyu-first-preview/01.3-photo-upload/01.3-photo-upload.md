## Overview

**Page Purpose:** 让小雨在上传或拍照前理解本次照片、历史结果和常用形象档案的边界，并完成一张当前照片的选择。

**User Situation:** 小雨想看真实预览，但担心照片被默认长期保存。她需要在动作发生前看到清楚、短句、可相信的照片使用说明。

**Success Criteria:** 用户上传或拍摄一张当前照片；用户理解“不保存为常用形象也可以生成”；用户知道可以重新选择照片。

**Entry Points:**
- `01.2-home-dual-entry` 点击“开始看看效果”。

**Exit Points:**
- 照片合格或仅有轻微提醒时，点击“用这张生成预览 / 仍然生成预览”直接创建默认生成任务并进入 `01.6-generation-waiting`。
- 严重照片问题、权限失败、读取失败或任务创建失败时停留本页。
- `01.4-photo-quality-check` 不再是首次 happy path 页面；其能力下沉为本页的 inline quality status 和异常分支。

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Empty | 未选择照片 | 显示上传区域和拍照/相册按钮 | 拍照或选择照片 |
| Permission denied | 用户拒绝权限 | 显示权限解释和系统设置引导 | 重新授权或返回 |
| Selected / Pass | 已选择可用照片 | 显示照片预览、通过状态和生成按钮 | 进入 `01.6` 或重新选择 |
| Selected / Warning | 照片可用但有轻微风险 | 显示 inline warning，不新增页面 | 仍然进入 `01.6` 或重新选择 |
| Selected / Fail | 照片严重不可用 | 显示阻断说明和重新上传 | 重新上传 |
| Upload error | 照片读取失败 | 显示具体错误 | 重选或稍后再试 |
| Check unavailable | 检查服务不可用但照片可读取 | 说明无法精确检查 | 先生成预览或重试检查 |
| Task submit error | 生成任务创建失败 | 说明未扣额度 | 重试生成或重选 |

## Technical Notes

- 当前 Expo app 已使用 `expo-image-picker`，可复用拍照/相册基础能力。
- 本页应作为可复用 Photo Intake：个人形象/发型建议与试穿场景共享拍照、相册、权限、预览、重选和错误恢复能力，由入口传入生成意图、照片要求和下一步目标。
- 本页不应默认创建常用形象。临时照片、本次生成结果和常用形象档案必须区分：照片可服务当前生成会话，结果进入历史，常用形象需要用户在结果后显式保存。
- 如果 MVP 暂不做服务端上传，仍需在文案上保持“本次照片”和“常用形象”边界。
- IMP-001 后首轮默认生成 intent 为 `daily_judgment_safe_direction`，source 为 `first_preview_auto_default`。
- 质量检查仍然存在为系统状态：`pass | warning | fail | unavailable`，但不作为独立 happy path 页面展示。

## Open Questions

| # | Question | Context | Status |
|---|----------|---------|--------|
| 1 | 照片是否在任务创建前上传到服务端，还是本地轻检后再上传？ | 影响隐私文案和错误处理。 | Open |
| 2 | 是否允许从历史照片复用？ | 首次 flow 目标是当前照片；历史复用可后续支持。 | Resolved: DD-001 不要求 |
| 3 | 个人形象建议和试穿是否共享拍照入口？ | 两类生成都需要当前照片，但后续照片要求和生成意图不同。 | Resolved: 共享 Photo Intake，入口传 intent/requirements/nextTarget |
| 4 | 临时照片是否等同于个人形象创建？ | 影响复用、保存和隐私边界。 | Resolved: 不等同；常用形象必须结果后显式保存 |
