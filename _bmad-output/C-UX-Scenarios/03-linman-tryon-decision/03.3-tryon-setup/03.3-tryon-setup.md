# 03.3-tryon-setup

## Overview

**Page Purpose:** 林曼确认本次参考对象、本人形象和生成消耗，确保开始生成前没有选错图、选错本人形象或误解额度扣除。

**Entry Context:** 林曼已经在 `03.2-import-reference` 导入了一个参考图。她需要在很短时间内确认“这就是我要试的对象”和“用这个本人形象生成”，然后开始生成。

**Exit Action:** 点击开始生成，进入 `03.4-tryon-waiting`。

**On-Page Interactions:**
- 查看参考图摘要，并可返回更换参考图。
- 查看常用本人形象是否可用，并可切换到“重新选择本人照片”的提示状态。
- 查看本次额度消耗、余额和失败不扣/失败处理说明。
- 选择试穿意图：衣服/发型/妆容，用于让生成设置更明确。
- 在资料不完整、余额不足、提交中或提交失败状态下看到不同的主行动反馈。

## User State

**Hope:** 她希望这一步足够短，只确认关键项，不重新配置复杂档案。

**Worry:** 她担心开始生成后才发现参考图或本人形象不对，也担心额度扣掉但没有可用结果。

## Functional Requirements

- 页面必须展示已导入参考图摘要，包含标题、来源和本次决策问题。
- 页面必须展示本人形象状态，默认使用常用本人形象。
- 页面必须展示本次消耗 `1 个生成额度` 和当前余额 `6`。
- 页面必须说明“开始生成后才消耗额度”以及失败处理预期。
- 主行动在 default 状态可进入 `03.4-tryon-waiting.html`。
- 页面必须支持 default、missing-profile、low-credit、submitting、submit-error 五种原型状态。
- 页面必须保持 Mobile-Only 单列布局，在 375px、393px、428px 宽度下无水平滚动。
