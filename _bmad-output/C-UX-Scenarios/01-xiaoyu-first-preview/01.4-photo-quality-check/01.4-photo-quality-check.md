## Overview

**Page Purpose:** 定义照片质量判断规则和异常恢复。IMP-001 后，本页不再作为首次 happy path 的独立页面；合格和 warning 状态应在 `01.3-photo-upload` inline 展示。

**User Situation:** 小雨已经选择了一张照片，但不知道它是否适合 AI 生成。如果失败原因不清楚，她会把责任归因于产品“不靠谱”。

**Success Criteria:** 合格照片进入生成设置；不合格照片获得明确重试建议；用户不会因为错误提示感到被评判。

**Entry Points:**
- 非首次 happy path 的详细检查、开发调试、或后续高级照片管理可进入。
- 首次 happy path 不应从 `01.3-photo-upload` 自动进入本页。

**Exit Points:**
- 照片合格或 warning 时，首轮应继续到 `01.6-generation-waiting`，而不是 `01.5-generation-setup`。
- 照片不合格时返回/停留 `01.3-photo-upload` 重新选择。

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Checking | 正在检查照片 | 显示检查中状态 | 等待，不重复提交 |
| Pass | 照片可用 | 显示通过摘要和继续按钮 | 继续选择风格 |
| Warning | 照片可用但有风险 | 显示提醒和“仍然继续” | 继续或重选 |
| Fail | 照片不适合 | 显示温和错误和重试建议 | 重拍或重选 |
| Check unavailable | 检查服务失败 | 不扣额度，说明可重试 | 重试检查或返回 |

## Technical Notes

- MVP 可先用前端或后端轻量规则检查：图片存在、尺寸、读取成功；更高级的人脸清晰度可后续接入。
- 如果质量检查只是模拟，文案不得承诺已经完成精确人脸识别。
- 检查失败不应消耗生成额度。
