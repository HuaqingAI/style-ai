## Component Naming Convention

**Format:** `[type]-[number]`

Examples:

- `icn-001` for icon components
- `brd-001` for brand components
- `btn-001` for buttons
- `inp-001` for input fields
- `crd-001` for cards

## Component File Structure

Each component file includes:

- Component ID
- Type and purpose
- Variants
- States
- Styling and tokens
- Usage tracking
