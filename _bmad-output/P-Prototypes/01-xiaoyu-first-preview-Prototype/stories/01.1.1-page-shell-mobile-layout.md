## What to Build

### HTML Structure

Create `01.1-start-login.html` in the prototype root with:

- Standard HTML document shell.
- Mobile viewport meta tag.
- Tailwind CDN for prototype speed.
- CSS variables for Style AI brand colors.
- A centered mobile app frame with min-height `100svh`.
- Four semantic placeholder regions:
  - brand trust header
  - login method panel
  - boundary note
  - legal links
- A small prototype-only status marker for current page and section.

Suggested structure:

```html
<body class="min-h-screen bg-[#111111] text-[#FAFAF7]">
```

## Common Issues & Fixes

### Issue: Horizontal Scroll

**Symptom**: Page scrolls sideways at 375px.  
**Cause**: A fixed-width child or unsupported max-width calculation.  
**Fix**: Remove fixed widths; use `w-full`, `max-w-[428px]`, and `box-sizing: border-box`.

### Issue: Content Too Tall On iPhone SE

**Symptom**: Legal/footer area pushes important future login content too far down.  
**Cause**: Vertical spacing too generous.  
**Fix**: Keep shell spacing moderate and let later sections use compact mobile typography.

### Issue: Overly Decorative Background

**Symptom**: Background competes with login content.  
**Cause**: Too many gradients or decorative accents.  
**Fix**: Use restrained brand black and subtle surface contrast only.

---

## Design Notes

Visual requirements:

- Keep cards and framed areas at 8px radius or less unless a later component has a stronger reason.
- Do not use decorative gradient orbs or bokeh backgrounds.
- Do not use purple as the dominant surface color; keep it as a restrained accent.
- Letter spacing should remain `0`.

UX considerations:

- This page is public-before-login and must feel low pressure.
- The shell must support the message that login is for quota/history/account management, not automatic long-term profile creation.
- The future primary action should remain reachable on small screens.
