## Overview

**Page Purpose:** 小雨看到第二次生成结果后，比较第一次和第二次方向，并保存一个她觉得有参考价值的结果，方便之后回看。

**Entry Context:** 她已经在 `02.1-second-style-exploration` 选择了一个第二次探索方向并完成生成。此刻她想确认这次结果和第一次是否真的有差异，以及哪一个更值得留下。

**Exit Action:** 点击保存结果，进入 `02.3-save-profile-prompt` 的常用形象提示。

**On-Page Interactions:**
- 查看第二次生成的结果预览和方向说明。
- 对照第一次结果与第二次结果，理解差异和可用价值。
- 在两个结果之间选择要保存的结果。
- 保存结果、查看已保存反馈，或在保存失败时重试。

## User State

**Hope:** 她希望把真正有参考价值的结果留下来，下次能回看比较，不用凭记忆做发型决定。

**Worry:** 她担心保存错结果、第一次和第二次差异不够明显，或者保存动作会让系统默认保存为常用形象。

## Functional Requirements

- 页面必须明确这是“第二次结果已生成后”的保存页。
- 页面必须展示第二次结果预览、选择方向、和第一次结果的差异说明。
- 页面必须提供第一次结果和第二次结果的轻量对照，让用户知道保存哪一个。
- 页面必须区分“保存到历史结果”和“保存为常用形象”，避免用户误解。
- 用户必须可以选择保存第一次或第二次结果，默认保存第二次结果。
- 主行动必须支持 default、saving、saved、save-error 四种状态。
- 保存成功后，主行动必须进入 `02.3-save-profile-prompt.html`；在该页未建成前可作为 forward placeholder。
- 页面必须保持 Mobile-Only 单列布局，在 375px、393px、428px 宽度下无水平滚动。
